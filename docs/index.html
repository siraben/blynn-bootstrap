Milestones

  1. Kernel starts with normal Linux boot messages on console=hvc0, then runs init=/init.
  2. /init prepares Alpine, materializes the TinyCC checkpoint into /work/blynn, and drops to a shell.
  3. blynn-tcc is available immediately from the Nix-built TinyCC checkpoint.
  4. Run bootstrap to verify the Nix-built riscv64 HCC checkpoint, then reproduce the TinyCC self-host checkpoint inside the VM.
  5. For the full proof run, use bootstrap --from-seed to rebuild stage0-posix, Blynn, HCC, and TinyCC from the seed path.
  6. After either build, the shell has the bootstrapped compiler as blynn-tcc and prints its SHA256 hash.
